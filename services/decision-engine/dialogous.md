# Диалоговые сценарии бота технической поддержки

## Архитектурные принципы

### Что делает бот
- Отвечает на вопросы
- Помогает найти информацию
- Проверяет статус
- Объясняет правила
- Подсказывает дальнейшие действия
- При необходимости переводит на оператора

### Что НЕ делает бот
- Не создаёт записи
- Не переносит брони
- Не инициирует возвраты
- Не управляет расписанием
- Не изменяет данные аккаунта
- Не принимает финансовые решения

### Минимальный набор действий бота

**Базовые:**
- `provide_information` - предоставить информацию из базы знаний
- `clarify_request` - уточнить запрос пользователя
- `request_identifier` - запросить идентификатор (номер записи, телефон, etc)
- `show_status` - показать статус объекта
- `suggest_solution` - предложить решение из базы знаний
- `provide_instruction` - дать инструкцию по действию
- `escalate_to_operator` - перевести на оператора

**Опциональные:**
- `show_contact_information` - показать контактную информацию
- `return_to_menu` - вернуться в главное меню

## 2. Записи и бронирование

### 2.1 Что умеет бот
- Показать информацию о записи
- Объяснить правила отмены/переноса
- Подсказать как изменить запись
- Сообщить статус записи
- Перевести на оператора

### 2.2 Типовой сценарий: узнать информацию о записи

**Пользователь:** Хочу узнать про запись

**Бот:**
1. Запрашивает идентификатор:
   - номер записи
   - или номер телефона

2. Если запись найдена - показывает:
   - дата и время
   - мастер
   - услуга
   - статус

3. Предлагает:
   - правила отмены
   - контакты
   - связаться с оператором

**Если запись не найдена:**
- Предлагает проверить введённые данные
- Предлагает обратиться к оператору

### 2.3 Сценарий: правила отмены и переноса

**Пользователь:** Как отменить запись?

**Бот:**
- Отмена возможна не позднее чем за 24 часа до записи
- Для отмены свяжитесь с оператором или воспользуйтесь сайтом
- [предлагает связаться с оператором]

**Пользователь:** Можно ли перенести запись?

**Бот:**
- Перенос возможен не позднее чем за 24 часа до записи
- Для переноса свяжитесь с оператором
- [предлагает связаться с оператором]

### 2.4 Что НЕ делает бот в этом сценарии
- Не создаёт записи
- Не переносит записи
- Не подтверждает записи
- Не выбирает слоты
- Не подбирает мастера
- Не ищет альтернативы
- Не выполняет автоматическую отмену

## 4. Оплата

### 4.1 Что умеет бот
- Показывает статус платежа
- Объясняет причины ошибок
- Даёт инструкции
- Переводит на оператора

### 4.2 Сценарий: деньги списались, но услуга не активирована

**Пользователь:** Деньги списались, но запись не появилась

**Бот:**
1. Запрашивает ID платежа или номер записи
2. Проверяет статус:
   - Платёж найден, в обработке:
     - Подождите 5-10 минут, обработка может занять время
     - Проверьте почту - должно прийти подтверждение
   - Платёж успешен:
     - Если запись не создана, свяжитесь с оператором
   - Платёж не найден:
     - Проверьте данные или обратитесь в банк
3. Предлагает:
   - подождать
   - проверить почту
   - связаться с оператором

### 4.3 Сценарий: оплата не прошла

**Пользователь:** Оплата не проходит

**Бот:**
Возможные причины:
- Недостаточно средств

- Неверные данные карты

Решения:
- Проверьте баланс карты
- Попробуйте другую карту
- Обратитесь в банк
- [предлагает оператора]

## 5. Проблемы с сайтом или приложением

### 5.1 Что умеет бот
- Классифицирует проблему
- Предлагает базовые решения
- Собирает минимум информации
- Эскалирует при необходимости

### 5.2 Сценарий: не работает сайт

**Пользователь:** Не работает сайт

**Бот:**
1. Уточняет:
   - Какое устройство (компьютер/телефон)
   - Какой браузер
   - Какая ошибка (текст или скриншот)

2. Предлагает базовые решения:
   - Обновить страницу (F5 или Ctrl+R)
   - Очистить кэш браузера
   - Попробовать другой браузер
   - Проверить подключение к интернету

3. Если не помогло:
   - Собирает информацию для тикета
   - Передаёт оператору

### 5.3 Сценарий: не могу войти

**Пользователь:** Не могу войти в аккаунт

**Бот:**
1. Уточняет проблему:
   - Забыл пароль
   - Не приходит код подтверждения
   - Ошибка при входе

2. Предлагает решения:
   - Воспользоваться функцией "Забыл пароль"
   - Проверить папку SPAM в почте
   - Попробовать другой браузер
   - Подождать и попробовать позже

3. Если проблема сохраняется:
   - [предлагает оператора]

### 5.4 Что НЕ делает бот в этом сценарии
- Не создаёт тикеты
- Не выполняет сложную диагностику
- Не выполняет многошаговый troubleshooting

**Пользователь:** Забыл пароль

2. Инструкция:
   - Нажмите "Забыл пароль" на странице входа
   - Введите ваш email или телефон
   - Следуйте инструкциям в письме/SMS

3. Если не помогает:
   - [предлагает оператора]

## Архитектурные преимущества

### 1. Бот становится knowledge-assistant
Вместо: "support crm automation system"
Получается: "faq + routing assistant"

### 2. Сильно упрощается state machine
Вместо: "booking orchestration"
Получается: "intent → answer → escalation"

### 3. Уменьшается количество интеграций
Не нужны:
- Календарные операции
- Транзакционные операции
- Orchestration flows
- Rollback логика
- Конфликтное состояние

### 4. Упрощается decision-engine
Вместо:
```json
{
  "intent": "reschedule_booking",
  "actions": ["load_slots", "validate_time", "reserve_slot", "notify_master"]
}
```

Получается:
```json
{
  "intent": "booking_question",
  "action": "provide_information"
}
```

### 5. Что остаётся для диплома
- Intent classification
- Dialogue transitions
- Context handling
- FAQ
- Escalation
- Базовая интеграция с backend
- Retrieval из knowledge base

Это полноценный support assistant без необходимости строить CRM/ERP orchestration систему.
